autocom.rpa.aa.exebot(path_driver, username, password, bot_name, base_aa_url, path_bot, device_run='', descrip_run='')

Este método ejecuta un BOT de Automation Anywhere, ingresa al Control Room de AA, busca el BOT indicado e inicia su ejecución. Este módulo requiere el módulo selenium, el cual a su vez requiere la instalación del controlador para Chrome. (Ver: https://chromedriver.chromium.org/).

Args:

  • path_driver (str): Path donde se localiza el driver necesario para controlar el browser Chrome mediante el módulo: selenium

  • username (str): Nombre de usuario del acceso al Control Room de AA.

  • password (str): Contraseña del acceso al Control Room de AA.

  • bot_name (str): Nombre del BOT que se ejecutará.

  • base_aa_url (str): URL del Control Room de AA donde se encuentra el BOT a ejecutar (p.e. https://community2.cloud-2.automationanywhere.digital/#/)

  • path_bot (str): Ubicación del BOT que se ejecutará separado por diagonales (p.e. /Bots/Subcarpeta)

  • device_run (str, optional): Device name del dispositivo que se utilizará para ejecutar el BOT, si se envía una cadena vacía se utiliza el disposiivo predeterminado. Defaults to «».

  • descrip_run (str, optional): Descripción opcional a capturar en la ejecución del BOT. Defaults to «».
