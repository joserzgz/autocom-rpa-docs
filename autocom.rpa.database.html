class autocom.rpa.database.DatabaseConnection(connection_string)

Bases: object

Clase que maneja la conexión a una base de datos y proporciona los métodos necesarios para ejecutar comandos SQL en dicha base de datos. Se utiliza ODBC para estandarizar la conexión a las bases de datos, por lo que es necesario que se instale el controlador ODBC necesario para conectarse a la base de datos destino.

close()

Método para cerrar la conexión hacia la base de datos.

read_to_dataset(sqlquery, encoding='utf8')

Realiza una consulta a la base de datos mediante el comando en «sqlquery», regresa el conjunto de registros resultante en formato JSON.

Args:

  • sqlquery (str): Comando de consulta SQL a realizar.

  • encoding (str, optional): Tipo de codificación de caracteres con el que se regresan los registros. Defaults to «utf8».

Returns:

  • array de json: Arreglo de registros en formato JSON del resultado de la consulta.

autocom.rpa.database.connect_ssh_tunnel(ssh_tunnel_config)

Realiza la conexión de un tunel SSH hacia el servidor.

Args:

  • ssh_tunnel_config (JSON object): Objeto JSON con los parámetros para establecer la conexión.

Los parámetros requeridos en el JSON son:

{
    ssh_keyfilename: "<str>",
    ssh_hostname: "<str>",
    ssh_port: <int>,
    ssh_username: "<str>",
    remote_address: "<str>",
    remote_port: <int>
}

  • ssh_keyfilename: Path donde se ubica el certificado PEM para realizar la autenticación al conectarse al tunel SSH.

  • ssh_hostname: URL o IP hacia donde se conectará el tunel SSH.

  • ssh_port: Puerto a utilizar para la conexión SSH. Por default 22.

  • ssh_username: Nombre de usuario para establecer la conexión SSH.

  • remote_address: IP destino de la conexión.

  • remote_port: Puerto destino de la conexión. (p.e. para una conexión para el servidor MySql por default es 3306).

Returns:

  • SSHTunnelForwarder: Regresa un objeto de la clase SSHTunnelForwarder de la conexión establecida.
